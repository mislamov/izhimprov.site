DNS-AID for izhimpro.ru
Date: 2026-06-11

Goal

Publish DNS for AI Discovery (DNS-AID) records for the public organization index of izhimpro.ru.

This repository exposes the HTTP discovery documents. The DNS-AID record itself must now be published in the authoritative Cloudflare DNS zone for `izhimpro.ru`.

Current authoritative DNS

As of 2026-06-11, the public zone is authoritative on Cloudflare:

- `joan.ns.cloudflare.com`
- `rommy.ns.cloudflare.com`

Current HTTP discovery endpoints

- Organization index: `https://izhimpro.ru/.well-known/agents-index.json`
- Site description: `https://izhimpro.ru/.well-known/site-description.json`
- API catalog: `https://izhimpro.ru/.well-known/api-catalog`
- Service description: `https://izhimpro.ru/.well-known/service-desc.json`
- MCP server card: `https://izhimpro.ru/.well-known/mcp/server-card.json`
- A2A agent card: `https://izhimpro.ru/.well-known/agent-card.json`
- Agent skills index: `https://izhimpro.ru/.well-known/agent-skills/index.json`
- Auth.md: `https://izhimpro.ru/auth.md`

Current state

- HTTP discovery on the website is working through Cloudflare.
- Homepage `Link` headers are published.
- `/.well-known/api-catalog` is published with `application/linkset+json`.
- `isitagentready` reports `https://izhimpro.ru/` at level `5` (`Agent-Native`) for HTTP-facing checks.
- DNS-AID now resolves publicly for `_index._agents.izhimpro.ru`, but DNSSEC validation is not yet confirmed by scanners.

Recommended DNS-AID record

Use an HTTPS record at the well-known owner name `_index._agents.izhimpro.ru`.

`key65400` is used as a private-use `SvcParamKey` for the HTTP path of the organization index until DNS-AID keys are registered by IANA.

Multiline form:

_index._agents.izhimpro.ru. 3600 IN HTTPS 1 izhimpro.ru. (
  alpn="h2,h3"
  port=443
  mandatory=alpn,port,key65400
  key65400="/.well-known/agents-index.json"
)

Single-line form:

_index._agents 3600 HTTPS 1 izhimpro.ru. alpn="h2,h3" port=443 mandatory=alpn,port,key65400 key65400="/.well-known/agents-index.json"

Why only _index._agents

- The site currently publishes an organization-level machine-readable index.
- Read-only `a2a` and `mcp` HTTP endpoints now exist on `https://izhimpro.ru/a2a` and `https://izhimpro.ru/mcp`.
- Separate `_a2a._agents` or `_mcp._agents` DNS publication is still optional and should only be added if dedicated DNS-AID protocol entrypoints are needed.

DNSSEC requirement

DNS-AID is materially stronger with DNSSEC. Enable DNSSEC for the public zone in Cloudflare and confirm that the parent zone publishes a `DS` record for `izhimpro.ru`.

Minimum checks after enabling DNSSEC:

- `_index._agents.izhimpro.ru` HTTPS returns the HTTPS RRset
- `izhimpro.ru` DS returns a DS record in the parent zone
- `_index._agents.izhimpro.ru` `RRSIG` returns signatures for the discovery RRset

Validation

Recheck after DNS publication:

- `https://isitagentready.com/`

Expected result:

- DNS-AID well-known entrypoint is found
- DNSSEC-backed validation succeeds for the published discovery zone
- The remaining `isitagentready` DNS-AID failure clears once DNSSEC validation succeeds

Cloudflare implementation note

The current zone is on the Cloudflare Free plan. The DNS-AID publication must therefore use only free Cloudflare DNS capabilities.

Operational note

Before the nameserver cutover to Cloudflare, `_agents.izhimpro.ru` and `_index._agents.izhimpro.ru` were previously published outside the main zone. After the cutover, those records are no longer publicly visible and must be recreated in the current authoritative DNS path.
